# SketchUp Demo Recorder вҖ” Design

## Purpose

Record user clicks/keys in SketchUp, edit per-event pause timings in a visual timeline, replay the sequence while screen-recording the SketchUp viewport вҖ” to produce demo videos repeatably.

## Stack

| Layer | Tech |
|---|---|
| Event capture / replay / window mgmt / hotkeys | Hammerspoon (Lua) |
| UI | HTML/JS inside `hs.webview` |
| Sequence storage | JSON files |
| Viewport resize on SU side | Mini Ruby companion plugin (file-IPC) |
| Screen capture | `screencapture -V` (or ffmpeg) via `hs.task` |

## Modules (Lua)

| File | Responsibility |
|---|---|
| `init.lua` | Wire everything, register hotkeys, menubar icon |
| `recorder.lua` | Global `hs.eventtap` вҶ’ append events to active sequence |
| `replayer.lua` | Post mouse/keyboard events with `pause_before_ms` delays |
| `store.lua` | Load/save JSON sequences |
| `companion_bridge.lua` | Write `/tmp/sdr_cmd.json`, read `/tmp/sdr_result.json` |
| `window_sizer.lua` | Apply viewport mode (companion) or window mode (`hs.window`) |
| `screen_capture.lua` | Start/stop `screencapture -V` with region |
| `ui.lua` | Build `hs.webview`, JS вҶ” Lua bridge |

## SU Companion Plugin

`sdr_companion.rb` in `~/Library/Application Support/SketchUp 2026/SketchUp/Plugins/`.

- `UI.start_timer(0.2)` polls `/tmp/sdr_cmd.json`
- Commands: `resize_viewport`, `get_window_bounds`, `ping`
- Writes result to `/tmp/sdr_result.json`
- Tiny (~80 lines), follows project SketchUp Extension Warehouse rules from `~/Code/skp/CLAUDE.md`

## Sequence JSON Schema

```json
{
  "version": 1,
  "name": "fair-align-3cubes",
  "created_at": "2026-05-16T12:00:00Z",
  "viewport": {
    "mode": "viewport",
    "preset": "youtube_1080p",
    "width": 1920,
    "height": 1080,
    "region": { "x": 100, "y": 200, "w": 1920, "h": 1080 }
  },
  "chrome_offsets": { "top": 70, "bottom": 25, "left": 0, "right": 0 },
  "events": [
    {
      "id": "evt_001",
      "type": "mouse_down|mouse_up|mouse_move|key_down|key_up|scroll",
      "button": "left|right|middle",
      "x": 540, "y": 320,
      "key": "z",
      "modifiers": ["cmd", "shift"],
      "scroll_dx": 0, "scroll_dy": -5,
      "pause_before_ms": 1000,
      "comment": "optional human note"
    }
  ]
}
```

- `pause_before_ms` вҖ” delay before event fires (not after). Set from inter-event delta on record; user edits in timeline.
- `mouse_down` and `mouse_up` are separate events (needed for drag).
- `mouse_move` throttled to ~30Hz on record (skip duplicate positions).

## Viewport Modes

| Mode | Behavior |
|---|---|
| `viewport` | Target WГ—H = SU model area. Tool calls `Sketchup.resize_viewport(model, w, h)` via companion. SU does chrome math itself. Tool queries final window bounds via companion to compute screen recording region (= window xy + viewport-in-window offset). |
| `window` | Target WГ—H = entire SU window. Tool uses `hs.window:setSize(w, h)`. Recording region = window bounds. |

## Presets (defaults)

- `youtube_1080p` 1920Г—1080
- `youtube_4k` 3840Г—2160
- `reels_1080p` 1080Г—1920
- `tiktok` 1080Г—1920
- `square_1080` 1080Г—1080
- `custom` user input

## User Flows

### A. Record
1. Pick mode + preset вҶ’ Apply (companion resize)
2. Manual: open SU scene, set start state
3. Hotkey вҢғвҢҘвҢҳR вҶ’ record
4. Click in SU вҶ’ events captured
5. Hotkey вҢғвҢҘвҢҳR вҶ’ stop вҶ’ timeline populated
6. Save

### B. Edit
- Click pause-chip вҶ’ inline edit ms
- Click event-chip вҶ’ edit/delete/duplicate/comment
- Drag to reorder
- Total duration recalc live

### C. Capture
1. Apply viewport
2. Manual: set SU start state
3. вҸә Record + Replay button:
   - Start `screencapture -V` on region
   - Wait 1s buffer
   - Replay events with `pause_before_ms`
   - Wait 1s tail
   - Stop capture вҶ’ mp4

## Hotkeys

| Combo | Action |
|---|---|
| вҢғвҢҘвҢҳR | Record clicks toggle |
| вҢғвҢҘвҢҳP | Replay sequence (no capture) |
| вҢғвҢҘвҢҳV | Show/hide tool |
| вҢғвҢҘвҢҳC | Record + Replay (capture) |

## Permissions Required

- macOS Accessibility (for `hs.eventtap` and `hs.window`)
- macOS Screen Recording (for `screencapture`)

## Out of Scope (V1)

- Recording audio
- Cross-platform (Windows/Linux) вҖ” macOS only
- App-aware event semantics (just OS-level)
- Cloud sync of sequences
